# AI 課綱比較頁內選課設計規格

## 背景與目標

目前使用者必須先在「候選課程」列表勾選比較課程，再前往「AI 功能」查看結果。這讓候選課程同時承擔排課與 AI 比較兩種任務，也讓使用者在進入比較工具後無法直接調整選擇。

本次調整將課程選擇完整移入「AI 功能 → AI 課綱比較」。候選課程頁回歸排課與管理用途；比較工具則在自己的頁面中提供搜尋、選取、限制提示與比較行動。

## 使用者流程

1. 使用者開啟「AI 功能」，選擇「AI 課綱比較」。
2. 頁面顯示目前所有候選課程與搜尋欄。
3. 使用者可依課名、教師或課號搜尋，再勾選 2 至 5 門課。
4. 頁面即時顯示已選數量、剩餘需求與五門上限。
5. 選滿至少兩門後，可選擇「使用 Gemini 比較」或「帶到 ChatGPT」。
6. 比較結果、個人化資料、API Key 與 ChatGPT 備援沿用既有流程。

## 候選課程頁

- 移除每門候選課程旁的比較核取方塊。
- 移除候選面板底部的比較工具列。
- 保留加入課表、詳細、課綱、鎖定、刪除、搜尋與篩選等原有功能。
- 候選課程頁不再直接啟動 Gemini 或 ChatGPT 比較。

## AI 課綱比較頁

### 課程搜尋與清單

- 在比較工具說明下方新增一個有可見標籤的搜尋欄。
- 搜尋範圍包含課名、教師與課號，忽略大小寫與前後空白。
- 清單顯示所有符合搜尋條件的候選課程。
- 每列至少顯示課名、教師、課號、學分與上課時間。
- 整列提供至少 44px 的可操作範圍；核取方塊保留原生語意與鍵盤操作。
- 搜尋只影響清單顯示，不會取消已選課程。

### 選取狀態

- 使用既有 `comparisonCourseIds` 作為單一選取狀態來源。
- 即時顯示「已選 N／5」。
- 少於兩門時，提示還需選幾門，並停用兩個比較按鈕。
- 選到五門後，尚未選取的課程停用；已選課程仍可取消。
- 提供「清除選取」，一次移除全部比較課程。
- 切換 AI 子功能、工作區分頁或返回 AI 功能首頁時保留選取。
- 候選課程被刪除後，自動從比較選取移除。

### 行動與結果

- 主要行動為「使用 Gemini 比較」。
- 次要行動為「帶到 ChatGPT」。
- 兩項行動都使用頁面內目前選取的 2 至 5 門課。
- Gemini Key 未設定時，沿用現有 API Key 教學與輸入流程，不清除選取。
- 比較載入、錯誤、重試、結果、官方來源與 ChatGPT 提示詞區維持既有功能。
- 個人背景、未來方向、學期目標與排課偏好仍為選填；有填寫時增加個人化建議，未填時提供客觀比較。

## 空白與錯誤狀態

- 沒有候選課程：顯示說明並提供前往「匯入／新增」的按鈕。
- 搜尋沒有結果：顯示「找不到符合的候選課程」，保留搜尋欄與已選狀態。
- 少於兩門：停用比較按鈕並說明剩餘數量。
- 已達五門：停用其他未選課程並說明需先移除一門。
- 課程遭刪除：重新整理比較清單與數量，不保留失效 ID。
- AI、官方課綱或網路錯誤：沿用既有可重試訊息，不清除搜尋與選取。

## 響應式與無障礙

- 桌面與手機皆採單欄搜尋與課程清單，不建立額外彈窗。
- 手機維持至少 44px 觸控目標，文字可完整換行且不得造成水平捲動。
- 搜尋欄使用可見標籤，核取方塊具有包含課名的可讀標籤。
- 選取數量與限制訊息透過 `aria-live="polite"` 宣告。
- 停用狀態同時使用原生 `disabled` 與可見文字說明，不只依賴顏色。

## 資料與架構

- 不修改 Worker 路由、AI 回覆格式、課綱抓取或 Gemini／ChatGPT 契約。
- `comparisonCourseIds` 仍為比較請求的唯一課程 ID 來源。
- 新增比較頁專用搜尋字串狀態，只控制顯示，不寫入長期儲存。
- 比較課程資料仍從目前 `courseStore` 解析，避免建立第二份課程資料。
- 候選課程渲染不再依賴比較狀態，降低兩個功能之間的耦合。

## TDD 驗收順序

1. **Red → Green → Refactor：候選頁去除比較控制**
   - 先建立會失敗的渲染測試，要求候選列不含比較核取方塊，且候選面板不含比較工具列。
   - 最小修改候選課程渲染與 HTML 結構，使測試通過後整理失效樣式與事件。
2. **Red → Green → Refactor：比較頁顯示可搜尋課程清單**
   - 先測試比較頁具備搜尋欄、完整課程資訊與課程核取方塊。
   - 實作最小渲染與搜尋行為，再整理命名與共用格式。
3. **Red → Green → Refactor：2 至 5 門限制與清除**
   - 先測試少於兩門、達五門與清除選取的真實互動狀態。
   - 實作限制、訊息與按鈕狀態，再整理狀態更新流程。
4. **Red → Green → Refactor：選取保留與刪除同步**
   - 先測試搜尋、分頁切換不清除選取，以及刪除候選課會移除失效選取。
   - 實作必要同步並重構為單一渲染入口。
5. **回歸與真實邊界驗證**
   - 執行所有單元、渲染、Worker、建置與 lint 測試。
   - 在桌面與手機寬度測試搜尋、選取 2–5 門、清除、Gemini 缺 Key、ChatGPT 備援與候選課程原有操作。

## 驗收標準

1. 候選課程頁不再出現任何比較核取方塊或比較工具列。
2. AI 課綱比較頁可搜尋課名、教師與課號。
3. 使用者只能選取 2 至 5 門進行比較，限制與恢復操作清楚。
4. 搜尋、切換分頁與返回 AI 功能首頁不會意外清除選取。
5. 刪除候選課程會同步移除比較選取。
6. Gemini、ChatGPT、API Key、個人化資料與結果顯示維持既有行為。
7. 桌面與手機介面可讀、可操作，沒有水平溢出或阻塞候選課程原有功能。
8. 完整自動化測試與瀏覽器回歸測試通過。
