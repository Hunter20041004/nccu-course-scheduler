# AI 課程比較設計規格

## 目標

讓學生從候選課程選取 2–5 門課，針對衝堂、內容重疊與個人目標取得可追溯的比較。網站內以使用者自己的 Gemini API Key 產生完整比較；沒有 Key 或想使用 ChatGPT 時，提供「複製提示詞並開啟 ChatGPT」的備援流程。

## 使用者流程

1. 候選課程每列提供「加入比較」核取方塊，最多選 5 門。
2. 選到 2 門後顯示比較工具列，列出已選數量、清除、`AI 比較` 與 `帶到 ChatGPT`。
3. `AI 比較` 使用目前分頁記憶體中的 Gemini Key。沒有 Key 時沿用既有 Key 教學與設定視窗。
4. 比較結果顯示在 AI 分頁，並分成：確定性衝堂、內容重疊、各課獨有價值、評量與負擔、客觀結論、個人化建議、資料限制與官方來源。
5. `帶到 ChatGPT` 由伺服器讀取相同的官方課綱、產生提示詞、複製到剪貼簿並開啟 ChatGPT；不承諾控制 ChatGPT 的輸入框。

## 個人化規則

- `自我介紹`、`未來方向`、`這學期想達成什麼`、`排課偏好`皆為選填。
- 四個欄位都未填時，仍產生客觀比較，結果明示「未使用個人化資料」。
- 有任一欄位時，AI 額外產生個人化建議，但必須與客觀比較分區，避免把偏好誤當成課綱事實。
- 比較工具列顯示非阻擋式提示：「建議先填寫目標與偏好，比對會更精準」，並能直接前往 AI 表單。
- 個人敘述只在請求時傳給 Gemini，沿用既有不在伺服器長期保存的政策。

## 資料與信任邊界

- 只允許伺服器抓取 `https` 且網域屬於 `nccu.edu.tw` 的官方課綱網址，避免任意網址存取。
- 每門課綱限制下載大小與擷取字數；移除 script、style、導覽與 HTML 標籤後才傳給模型。
- 課綱未上傳或暫時無法讀取時明確標示，不把「無法取得」寫成「沒有課綱」。
- 至少要有兩門可讀取的官方課綱才呼叫模型；不足時提供可操作的錯誤訊息與官方課綱連結。
- 衝堂由既有排課規則計算，AI 不負責判定時間衝突；AI 只解讀內容與使用者適配度。

## AI 回覆契約

Gemini 必須回傳結構化 JSON：

- `summary`：客觀總結。
- `overlap`：0–100 的重疊指標、程度標籤與共同主題。
- `courses`：每門課的重點、獨有價值、評量方式與負擔說明。
- `recommendation`：建議選擇的課程 ID、理由、信心程度；證據不足時可以不指定課程。
- `personalized`：是否實際使用個人資料，以及個人化理由。
- `limitations`：課綱缺漏、推論限制與不確定性。

所有課程 ID 必須來自輸入；未知 ID、缺少必要欄位或不合法數值都視為模型回覆錯誤。

## 介面設計

- 保留現有「雨後日光／輕夢核」視覺，不替比較功能建立另一套配色。
- 候選列的比較控制不搶走「點列加入課表」的主操作；核取方塊使用獨立 44px 觸控區，阻止事件冒泡。
- 比較工具列固定在候選面板底部但保留列表底部空間，避免遮住最後一門課。
- 比較結果使用緊湊的比較板：先顯示衝堂與結論，再顯示共同／差異內容；手機改為單欄，長文字可正常換行。
- 載入、成功、錯誤、課綱不足與未個人化皆有文字狀態，不只靠顏色表達。

## 錯誤處理

- 少於 2 門：停用比較按鈕並說明還需選幾門。
- 超過 5 門：保留原選取並提示上限，不自動移除其他課。
- Gemini Key 未設定：開啟既有 Key 視窗，保留比較選取。
- 課綱不足：不呼叫模型，列出哪些課無法取得及處理方式。
- AI 格式錯誤：以既有安全錯誤格式回傳，可重試但不洩漏 Key 或上游內容。
- 剪貼簿或彈出視窗失敗：在頁面顯示可手動複製的提示詞，不讓使用者遺失內容。

## 測試策略

- 單元測試：請求驗證、官方課綱 HTML 清理、比較回覆契約、ChatGPT 提示詞、個人化開關。
- Worker 測試：比較路由移除 Key、ChatGPT 備援不需要 Key、錯誤格式與 `no-store`。
- 真實邊界契約：至少一門 115-1 官方課綱能被下載並擷取課名與課程內容。
- 渲染測試：候選課比較控制、選取上限、個人化提示、結果區與 ChatGPT 備援存在於實際 bundle。
- 瀏覽器測試：桌面與手機完成選 2 門、AI 比較、清除、開啟 ChatGPT；確認原有加入課表、詳細、篩選與更多選單沒有失效。

## 非目標

- 不使用未公開或不穩定的 ChatGPT 預填網址。
- 不替使用者自動送出 ChatGPT 訊息。
- 不抓取非政大網域，也不長期保存完整課綱或個人敘述。
- 不讓 AI 取代既有衝堂、資格與開課狀態規則。
