# 首次使用導覽與完整教學中心設計規格

日期：2026-07-15

## 背景

這個工具要開放給朋友暫時公開試用。朋友多半沒有 API Key 使用經驗，也不熟悉政大排課工具的操作方式，因此需要在不阻斷主要排課流程的前提下，提供可跳過的首次導覽與可永久重開的完整教學中心。

目前網站已經有候選課程、課表、選課條件、實習設定、匯入/新增與 AI 推薦等功能。這次設計只新增教學與導覽層，不改變既有課程資料、排課規則或 AI 推薦規則。

## 目標

1. 第一次打開網站的使用者可以在 1 到 2 分鐘內理解主要操作。
2. 使用者可以跳過導覽，但首頁永久保留「使用教學」按鈕。
3. 完整教學中心要足夠詳細，讓沒用過 API Key 的朋友也能完成設定。
4. Gemini API Key 必須由使用者自己提供；網站不可內建或共用開發者的 Key。
5. API Key 只在使用者啟動 AI 功能時才要求，不在首次進站時強迫設定。
6. 教學與導覽不可修改課表、候選課程、選課條件或實習設定。

## 非目標

1. 不建立第三方導覽套件依賴，使用自製原生導覽。
2. 不建立登入系統。
3. 不保存使用者 API Key 到伺服器、資料庫、URL 或長期本機儲存。
4. 不新增新的公開網站專案；部署時更新既有公開測試網站。
5. 不用導覽功能重寫既有排課 UI。

## 整體結構

首頁或主要工具列固定提供「使用教學」入口。首次使用者會看到可跳過的精簡導覽入口，包含三個操作：

1. 開始快速導覽。
2. 直接開始使用。
3. 打開完整教學。

快速導覽完成或跳過後，不再於同一瀏覽器自動彈出。使用者仍可透過「使用教學」按鈕重新開啟完整教學，並從完整教學中重新開始快速導覽。

## 完整教學中心

完整教學中心是一個右側大型面板，不離開目前排課畫面。內容使用章節式導覽，讓使用者可以直接跳到需要的主題。

### 章節

1. 第一次使用
   - 說明工具用途。
   - 說明資料主要保存在使用者本機。
   - 說明不使用 AI 也可以手動排課。
   - 說明 API Key 關閉分頁即清除。

2. 取得 Gemini API Key
   - 到 Google AI Studio。
   - 登入 Google 帳號。
   - 建立 API Key。
   - 複製 Key。
   - 回到網站貼上。
   - 明確說明 Key 不會存到伺服器、資料庫或網址。

3. 匯入候選課程
   - 從政大課程追蹤清單截圖辨識匯入。
   - 手動新增課程。
   - 管理既有候選清單。
   - 說明沒有開的課不會匯入。
   - 說明有條件的課會進入選課條件。
   - 說明待確認課程需要自行檢查。

4. 管理選課條件
   - 說明系所、雙主修、學程、年級、身分限制的用途。
   - 說明為什麼有些課需要勾選條件才會顯示符合。
   - 說明條件不符合時課程仍可保留在候選清單，但應查看詳細資訊。

5. 排課操作
   - 點擊右側課程加入左側課表。
   - 說明「詳細」、「鎖定」、「刪除」三個按鈕。
   - 說明任何課程都可以鎖定，AI 方案會保留鎖定課程。
   - 說明一鍵清空目前課表。
   - 說明非同步課程會放在非同步工作區。
   - 說明實體、同步、非同步狀態可以切換。

6. 實習與個人行程
   - 設定實習天數。
   - 設定固定時段。
   - 設定彈性時段。
   - 新增社團、課外組織、打工、自修等個人行程。
   - 透過同一套衝堂檢查確認時間是否可行。

7. AI 推薦方案
   - 說明如何描述至少幾學分。
   - 說明如何描述想保留幾天實習。
   - 說明如何描述主題偏好。
   - 說明如何描述不想考試、不想早八、語文課偏好。
   - 說明鎖定課程會被保留。
   - 明確寫出：有衝堂的方案不應顯示；非同步課可以用來補學分，但仍需符合資格條件。

8. 常見問題
   - 為什麼 AI 說 API Key 沒設定？
   - 為什麼課程顯示條件不符合？
   - 為什麼截圖匯入後有待確認？
   - 為什麼 AI 推薦沒有某類課？
   - 為什麼套用方案後沒有排進課表？
   - 如何重跑新方案？
   - 如何清除自己的 API Key？

## 快速互動導覽

快速導覽採用自製原生 overlay。它只能切換頁籤、捲動、聚焦與說明，不得修改任何使用者資料。

### 視覺與互動

1. 使用半透明遮罩降低背景干擾。
2. 凸顯目前說明的 UI 區塊。
3. 說明卡靠近被凸顯的 UI。
4. 每一步提供「上一步」、「下一步」、「結束導覽」。
5. 手機或窄版畫面中，說明卡固定在底部。
6. 使用者按 Esc 或關閉按鈕時結束導覽。
7. 若目標元素不存在，導覽改用一般說明卡，不阻斷流程。

### 八個步驟

1. 課表區
   - 說明政大節次、週一到週日與衝堂檢查。

2. 候選課程
   - 切到「候選課程」tab。
   - 說明點擊課程可加入課表。

3. 課程按鈕
   - 聚焦「詳細」、「鎖定」、「刪除」。
   - 說明鎖定不限定必修課。

4. 選課條件
   - 切到「選課條件」tab。
   - 說明有條件的課會要求勾選身分或條件。

5. 實習設定
   - 切到「實習設定」tab。
   - 說明固定與彈性實習時段。

6. AI 推薦
   - 切到「AI 推薦」tab。
   - 說明如何填需求，以及何時需要 API Key。

7. 匯入/新增
   - 切到「匯入/新增」tab。
   - 說明截圖匯入、手動新增與待確認課程。

8. 完成檢查
   - 回到課表。
   - 說明套用方案前要看學分、衝堂、非同步與條件符合。

## API Key 與安全

1. 使用者自己的 API Key 只存在目前分頁的 sessionStorage 或記憶體。
2. 關閉分頁後 Key 必須清除。
3. 網站不可把 API Key 寫入 localStorage、伺服器、資料庫、URL 或部署設定。
4. 首次導覽不要求輸入 API Key。
5. 使用者按下 AI 推薦或截圖辨識等 AI 功能時，如果尚未設定 Key，才打開 Key 設定。
6. 完整教學中心必須提醒使用者：API Key 等同使用者自己的額度，請不要傳給別人。

## 狀態儲存

1. 快速導覽是否看過：localStorage。
2. 完整教學面板開關：目前頁面狀態，不需要長期記住。
3. API Key：sessionStorage 或記憶體。
4. 課表、候選課程、條件、實習設定：維持既有儲存方式。

## TDD 驗收策略

所有實作採垂直切片 Red -> Green -> Refactor，一次只新增或修改一個失敗測試。

1. 首次開站顯示導覽入口。
   - Red：新增測試，期望未看過導覽時顯示快速導覽入口。
   - Green：加入最小狀態與 UI。
   - Refactor：整理命名與初始化流程。

2. 跳過或完成導覽後不再自動顯示。
   - Red：新增測試，期望 localStorage 已標記時不顯示入口。
   - Green：接上 localStorage。
   - Refactor：抽出導覽狀態 helper。

3. 永久「使用教學」按鈕可開完整教學中心。
   - Red：新增測試，點擊按鈕後顯示教學中心。
   - Green：加入面板與開關。
   - Refactor：整理面板渲染。

4. 完整教學中心包含必要章節。
   - Red：新增測試，確認章節清單與關鍵安全提醒存在。
   - Green：填入教學內容。
   - Refactor：把章節資料結構化。

5. AI 功能沒有 Key 時才開 Key 設定。
   - Red：新增測試，確認首次載入不自動開 Key 設定；觸發 AI 功能才開。
   - Green：調整 Key dialog 觸發條件。
   - Refactor：整理 API Key 狀態流程。

6. 導覽可切 tab、聚焦目標，且不修改課表資料。
   - Red：新增測試，導覽下一步後 active tab 正確，課表資料未變。
   - Green：加入 tour controller。
   - Refactor：抽出步驟定義。

7. 窄版下導覽卡不遮住主要操作。
   - Red：新增渲染或樣式測試，確認窄版使用底部固定版型。
   - Green：加入 responsive CSS。
   - Refactor：整理樣式 token。

8. 完整驗證。
   - 執行 npm run verify。
   - 建置與部署到既有公開網站。

## 部署

1. 更新既有公開測試網站。
2. 不建立新網站。
3. 部署前必須先完成測試與 build。
4. 部署後回報公開網址，讓朋友可直接打開試用。

## 開放問題

無。上述規格已依序取得使用者確認。
