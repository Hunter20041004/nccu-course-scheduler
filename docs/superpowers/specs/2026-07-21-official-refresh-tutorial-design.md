# 官方資料更新教學設計規格

## 背景

候選課程現在支援重新抓取政大官方資料、保留學生已做的排課決定，並以三種狀態呈現課綱。但首次快速導覽與完整教學中心尚未說明這些行為，使用者容易把「老師尚未上傳」誤認為系統故障，也不知道重新查詢是否會清掉既有資料。

## 目標

1. 快速導覽說明已在候選清單的課程可使用「更新官方資料」。
2. 完整教學說明更新時會保留選課、鎖定、上課方式與班別安排。
3. 完整教學清楚區分「可以查看課綱」、「老師尚未上傳課綱」與「課綱狀態暫時無法確認」。
4. 常見問題提供失敗復原與成功後重新整理仍保留資料的答案。

## 呈現原則

- 沿用既有八步快速導覽與九章完整教學，不增加新的教學層級。
- 快速導覽只提供操作方向；完整定義與排錯集中在永久的教學中心。
- 每個錯誤狀態都同時說明原因與下一步，不只顯示狀態名稱。
- 不修改排課、官方資料更新、儲存或課綱判定邏輯。

## 驗收標準

- 快速導覽第三步包含「更新官方資料」與保留使用者決定的說明。
- 完整教學「加入候選」章節說明新課加入與既有課更新的差異。
- 完整教學「課程資料」章節逐一解釋三種課綱狀態與重新查詢方式。
- 「匯出與排錯」章節說明更新失敗不刪除舊資料，更新成功後資料可跨重新整理保留。
- 原有八步導覽、九章教學與所有排課功能測試維持通過。
