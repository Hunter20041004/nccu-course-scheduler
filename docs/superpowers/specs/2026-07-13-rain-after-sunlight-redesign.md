# 雨後日光排課系統修復與視覺改版

## 目標

修復正式私人網站中課表與 24 門候選課程未初始化的問題，並將既有排課系統改為「雨後日光」輕夢核視覺。所有既有功能與資料模型維持不變，完成後必須在使用者的 Chrome 中逐項實測。

## 根因與修復邊界

目前建置流程移除 ES module 的 `import`／`export` 後直接串接所有來源檔，使 `planner-core.mjs` 與 `internship-planner.mjs` 的私有 `overlaps` 函式落在同一頂層作用域。瀏覽器以 module script 解析時拋出 `SyntaxError: Identifier 'overlaps' has already been declared`，因此 `renderAll()` 從未執行。

建置器必須替每個來源模組建立獨立作用域，只公開明確列出的匯出名稱。測試必須以 ES module 規則解析最終內嵌腳本，防止一般函式解析測試漏掉頂層名稱衝突。

## 視覺系統

- 背景霧白 `#F5F3EF`，雲灰 `#D8D5D2`，主要文字深炭灰 `#454348`。
- 主要操作與選取狀態使用杏橘 `#FFAA55`。
- 實習保留時段使用雨後藍 `#6879C9`。
- 選修與非同步課程使用霧紫 `#9180B5`。
- 衝堂與錯誤使用警示紅 `#D94A48`。
- 面板使用半透明霧面白、柔和擴散陰影、低對比顆粒與少量橘／藍／紫光球；不得使用常見藍紫漸層背景。
- 政大方格課表保持清楚、方正、可掃讀；裝飾不得壓過節次、課名與操作狀態。
- 互動動畫限 200ms 內的淡入、顏色與按壓回饋，並遵守 `prefers-reduced-motion`。

## 介面與互動

- 左側維持政大官方 16 節次課表；右側維持 24 門候選課程。
- 必修、選修、實習、衝堂與非同步使用顏色加文字／圖樣雙重識別，不以顏色作為唯一訊號。
- 搜尋、篩選、課程加入與移除、多課號／多老師選擇、非同步切換、三種快速方案、資格條件、實習自動與固定模式、手動新增、截圖交接、重設與本機保存都必須維持可用。
- 桌機維持左右雙欄；窄螢幕改為單欄，政大課表可在自己的容器內水平捲動。
- 所有可操作元素至少 44px，保留鍵盤焦點樣式與可讀對比。

## 驗證

1. 先以失敗測試重現最終 module script 的重名語法錯誤，再修正建置器。
2. 以成品 HTML 契約測試鎖定雨後日光色票、夢核背景層、減少動態效果與既有核心控制。
3. 執行全部單元、成品、語法與建置測試。
4. 私人部署成功後，在 Chrome 逐項操作核心流程，並檢查桌機與 375px 手機寬度、瀏覽器錯誤紀錄及狀態持久化。
