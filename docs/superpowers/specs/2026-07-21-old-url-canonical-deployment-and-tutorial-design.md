# 舊網址恢復為唯一正式站與教學補全設計規格

## 目標

將 `https://nccu-course-planner-1151.huntertseng.chatgpt.site/` 恢復為 Sunbreak 政大排課工具的唯一正式入口，並把目前 `main` 上所有新功能、網站內導引、GitHub 作品集連結與發布設定統一指向這個網址。

同時補全最近新增功能的使用教學，讓第一次使用的政大學生能從空白候選清單開始，完成官方課程搜尋、排課、資格檢查、AI 課綱比較、AI 推薦與跨裝置搬移資料。

## 成功條件

1. 舊網址部署目前 `main` 的完整功能，不再停留於舊版本。
2. 網站內所有「完整版」或「正式站」連結只指向舊網址。
3. GitHub README 與作品集測試只把舊網址列為正式展示網址。
4. `.openai/hosting.json` 綁定舊網址所屬 Sites 專案，不再發布到替代網址。
5. 使用教學涵蓋最新 AI 功能、官方課程資料與資料搬移限制。
6. 發布不清除或覆寫使用者已保存在舊網址瀏覽器中的候選課程與課表。

## 使用者資料與網址切換

排課資料保存在瀏覽器的網站專屬空間。可以把它想成每個網址各有一個獨立抽屜：更新同一個舊網址不會清空抽屜，但另一個網址的資料不會自動搬回來。

- 原本就在舊網址排課的使用者：發布後保留同一瀏覽器中的資料。
- 曾在替代網址排課的使用者：需要先匯出排課 JSON，再到舊網址匯入。
- API Key、AI 個人資料、原始截圖：不寫入長期儲存，也不包含在排課 JSON 中。

## 教學補全

### 快速導覽

保留八步原生導覽，並確保 AI 步驟清楚說明：

1. AI 排課推薦與 AI 課綱比較位於同一個「AI 功能」分頁。
2. 兩項功能共用一份選填個人資料。
3. 課綱比較要先在比較頁選擇 2–5 門候選課程。
4. 重新整理會清除 API Key 與 AI 個人資料。

### 完整使用教學

在既有任務導向教學中補明以下內容：

1. **AI 課綱比較的前置條件**：欲比較的課程必須先加入候選清單；比較頁可搜尋課名、教師或課號並選 2–5 門。
2. **官方課綱限制**：沒有上傳課綱或官方課綱暫時讀不到時，系統要明確標示資料限制，不得由 AI 補造課程內容。
3. **共用個人資料**：自我介紹、未來方向、學期目標與排課偏好在推薦與比較之間即時互通；均為選填，空白時仍提供客觀比較。
4. **隱私與重新整理**：API Key 與共用個人資料只存在目前分頁，重新整理或關閉分頁即清除。
5. **跨裝置與換網址**：排課資料只存在原本網址與瀏覽器；使用 JSON 匯出與匯入搬移候選課程、課表、鎖定、資格與實習設定。
6. **官方課程資料**：說明官方搜尋、更新官方資料、課綱三種狀態、選課資格、非同步與多時段處理。

## TDD 實作順序

每個變更遵循一次一個垂直切片的 Red → Green → Refactor：

1. 先讓作品集測試要求舊網址為唯一正式站，確認失敗；再修改 hosting、README 與程式常數使其通過。
2. 再讓頁面測試要求 AI 比較前置條件與官方課綱限制教學，確認失敗；再補上最少教學文字使其通過。
3. 再讓頁面測試要求跨網址 JSON 搬移說明，確認失敗；再補上最少教學文字使其通過。
4. 每個切片通過後整理重複文字與命名，並再次確認全綠。
