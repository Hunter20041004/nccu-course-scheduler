# 任務導向使用教學更新設計規格

## 背景

Sunbreak 已新增政大課程庫搜尋、候選課程時間摘要、完整課程詳細資料、官方課綱、可切換的上課形式、候選課程清空與手機桌布匯出等功能。現有首次提示、八步快速導覽與完整教學中心沒有完整涵蓋這些操作，第一次使用的學生容易知道「有功能」，卻不知道應依什麼順序完成排課。

## 目標

1. 讓第一次使用者依照實際任務順序完成第一份課表。
2. 同步更新首次提示、八步快速導覽與完整教學中心，避免三處說明互相矛盾。
3. 清楚區分不需要 API Key 的一般排課與需要個人 Gemini API Key 的 AI 功能。
4. 說明目前所有主要操作，包括課程來源、詳細資料、課綱、修課條件、上課形式、鎖定、清除、實習、AI 推薦與桌布匯出。

## 非目標

- 不改變排課、資格判斷、AI 推薦、儲存或匯出邏輯。
- 不新增新的教學框架、分析追蹤或跨裝置同步。
- 不在教學過程自動新增、刪除、鎖定或選取任何課程。

## 教學資訊架構

### 首次提示

首次提示保留三個入口：開始快速導覽、直接開始使用、打開完整教學。主要說明改成四步心智模型：

1. 搜尋或匯入課程。
2. 加入候選清單。
3. 檢查時間與修課條件。
4. 加入課表並處理衝堂。

提示同時說明一般排課不需要 API Key，只有截圖辨識與 AI 推薦需要使用者自己的 Gemini API Key。

### 八步快速導覽

快速導覽沿用現有原生聚光燈與頁籤切換機制，不修改使用者資料。八個步驟依序為：

1. **課表總覽**：認識政大節次、週一至週日、實習保留與排課提醒。
2. **候選課程**：查看課名、教師、課號、學分、時間與資格狀態；點課程加入或移出課表。
3. **詳細與管理**：說明「詳細」、「課綱」、「鎖定」、「刪除」，以及詳細資料只展開一門。
4. **選課條件**：勾選自己的系所、雙主修、學程、年級或其他身分條件，並閱讀條件影響。
5. **實習設定**：設定目標天數、固定或自動時段，理解實習區塊與課程衝突。
6. **AI 推薦**：輸入背景、學期目標與偏好；說明最低學分、無衝堂、非同步和鎖定課的影響。
7. **匯入與新增**：介紹政大課程庫、AI 截圖辨識、手動課程、社團、組織與個人行程。
8. **最後檢查與匯出**：檢查學分、資格、上課形式、衝堂、非同步課與提醒，再匯出手機桌布。

### 完整教學中心

完整教學中心改為九個任務章節：

1. **第一次開始排課**：資料從空白開始、瀏覽器本機保存、不同網址與裝置不共享資料、快速導覽不修改資料。
2. **加入候選課程**：逐步說明政大 115-1 課程庫搜尋、AI 截圖辨識與手動新增三種方式，以及待確認狀態。
3. **查看課程資料**：說明候選列的時間摘要、行內完整詳細資料、官方備註、班別時段和官方課綱。
4. **設定修課條件**：說明條件如何產生、如何勾選、資訊性備註與真正限制的差異，以及「條件不符合，請看詳細。」的處理方式。
5. **建立與管理課表**：說明點課程加入／移出、鎖定、刪除、一鍵清空課表和一鍵清空候選清單的差異。
6. **上課形式與衝堂**：說明「實體／同步／非同步」選擇、非同步不占每週固定時段、同步活動與考試仍可能衝堂，以及多時段課程的安排選擇。
7. **實習與其他行程**：說明目標天數、固定／自動實習時段，以及新增社團、課外組織、打工、自修或個人行程。
8. **AI 推薦與 API Key**：說明如何申請 Gemini API Key、分頁關閉即清除、輸入有效需求、三個方案的預覽／套用、鎖定課保留、最低學分與無衝堂原則。
9. **匯出與常見問題**：說明手機桌布匯出內容，以及 API Key、待確認、跨裝置資料、清除資料、靜態分享版與完整站的常見問題。

## 呈現方式

- 沿用現有 `dialog` 教學中心、頂部分章導覽與正文獨立捲動。
- 每章以「操作步驟」、「你會看到什麼」和必要的「注意」組成，避免只有單段抽象描述。
- 重要安全或破壞性操作使用現有警示色文字，不另外增加裝飾性視覺。
- 桌機保持可掃讀的雙欄步驟卡；手機退化為單欄，所有互動目標至少 44×44px。
- 章節導覽在手機可橫向捲動，不遮住正文。

## 可及性與互動

- 保留原生 `dialog`、標題關聯、關閉按鈕標籤與鍵盤焦點行為。
- 章節連結仍以錨點導覽，正文標題維持階層順序。
- 快速導覽的每個步驟都必須有可見聚光燈；找不到目標時仍顯示說明卡，但不修改資料。
- 教學文字不只依賴顏色傳達狀態。

## 資料與安全說明

- 課表與候選資料只保存在目前網站來源的瀏覽器儲存空間。
- 換瀏覽器、換裝置、清除網站資料，或使用不同正式站網址，都不會自動共享原資料。
- Gemini API Key 只保留在目前分頁記憶體，關閉或重新整理即清除；不可分享給其他人。
- 政大課程庫搜尋與一般排課不需要 API Key。

## 測試策略

依 TDD 垂直切片執行：

1. **完整教學內容**：先寫失敗測試，驗證九個章節及所有新功能關鍵詞，再更新 HTML 使其通過。
2. **首次提示與快速導覽**：先寫失敗測試，驗證四步心智模型與新版八步內容，再更新提示和步驟定義。
3. **教學版面**：先寫失敗測試，驗證桌機步驟卡、手機單欄與 44px 目標，再加入最少 CSS。
4. 執行完整單元、建置、HTML、語法與政大真實 API 契約測試。
5. 用 Chrome 測試首次提示、八步聚光燈、教學章節導覽、桌機與 390px 手機版面，確認無水平溢出與主控台錯誤。

## 驗收標準

- 首次提示、快速導覽與完整教學使用一致的任務順序和術語。
- 完整教學包含九個章節，涵蓋目前所有主要功能。
- 使用者能分辨清空課表與清空候選清單的不同影響。
- 使用者能理解修課條件、上課形式、非同步與衝堂的關係。
- 使用者能在不知道 API 的情況下完成 Key 申請、安全貼上與 AI 功能操作。
- 快速導覽八步都有聚光燈，且不會修改排課資料。
- 390px 手機寬度無水平溢出，教學內容與按鈕可完整操作。
