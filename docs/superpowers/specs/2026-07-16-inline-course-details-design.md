# 候選課程列內完整詳情設計

## 目標

將候選課程目前會被容器裁切的浮動詳情卡，改為課程列內的全寬展開面板。使用者點擊「詳細」後，能在原列表位置閱讀該課程所有已知資料，不遮住其他課程，也不超出右側工作區。

## 互動

- 「詳細」維持 44×44px 的觸控目標，使用原生 `details`／`summary` 語意與展開狀態。
- 展開內容位於該候選課程主列與操作列的正下方，橫跨整個候選課程寬度。
- 同一時間只展開一門課。開啟另一門時，先關閉原本的詳情。
- 展開後保留在候選列表的正常捲動流程中；不得使用絕對定位、浮層或固定高度裁切內容。
- 收合後恢復緊湊候選列，不影響一次瀏覽多門課的密度。

## 詳情內容

依照目前資料完整度顯示：

1. 課程基本資料：課名、課號、教師、學分。
2. 時間與形式：政大星期／節次摘要、可非同步狀態。
3. 資格狀態：條件符合、需確認、條件不符合或未開課；非符合狀態同時顯示原因。
4. 官方備註／選課條件：完整保留所有文字，不做省略或單行截斷。
5. 官方班別／多時段選項：逐項完整顯示。
6. 來源資訊：政大課程庫、AI 匯入、手動新增或內建資料。
7. 官方課綱：只有可信任的 NCCU HTTPS 網址才提供「開啟官方課綱」連結；沒有網址時顯示「目前沒有可用的官方課綱」。

## 視覺與響應式

- 面板使用淡紫底、細邊框與清楚的資訊區塊層級，延續目前 Sunbreak 配色。
- 基本資料使用自適應網格；桌面可多欄、手機自動改為兩欄或單欄。
- 長文字允許自然換行；不使用 `text-overflow: ellipsis`。
- 手機 375–390px 寬度不得產生頁面水平捲動；內容與操作目標不得被裁切。

## 錯誤與空資料

- 缺少教師、課號或其他欄位時顯示「未提供」，避免空白造成誤解。
- 沒有額外限制時明確顯示「目前沒有額外限制」。
- 詳情面板只呈現既有／已驗證資料，不推測或補造課程資訊。

## 測試

採 TDD 垂直切片：

1. 先以失敗測試要求詳情面板位於課程列內並橫跨全寬，再移除絕對定位使其通過。
2. 先以失敗測試要求完整資料欄位與不省略長文字，再補齊結構化內容。
3. 先以失敗測試要求一次只展開一門，再加入互斥收合邏輯。
4. 跑完整單元、介面與政大契約測試，最後使用 Chrome 驗證桌面與 390px 手機版。
