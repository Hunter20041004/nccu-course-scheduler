# 新訪客空白工作區與政大課程庫搜尋設計

日期：2026-07-16  
狀態：已由使用者確認

## 目標

讓公開網站適合不同學生各自開始排課：新訪客不再看到網站作者的候選課程或課表，但既有使用者在同一網址、同一瀏覽器中保存的資料仍會復原。使用者可從政大 115-1 官方課程庫、AI 截圖辨識或手動輸入建立自己的候選課程，並能一次清空候選清單。

## 已選方案

採用瀏覽器直接查詢政大官方課程介面。實際契約驗證顯示 `https://es.nccu.edu.tw/course/` 的 115-1 查詢可用，且回應包含 `Access-Control-Allow-Origin: *`，因此 GitHub Pages 靜態版也能搜尋，不需要 Gemini API Key 或本站後端代理。

不採用的替代方案：

- 本站後端代理：可集中快取與控管錯誤，但增加部署、流量與單點故障，而且 GitHub Pages 仍需另一個 API 服務。
- 定期匯入整份靜態課程庫：讀取快且不依賴即時官方服務，但容易過期，也需維護更新工作。

## 初始狀態與既有資料

### 新訪客

當排課儲存鍵不存在時：

- 候選課程為空陣列。
- 已排課程為空陣列。
- 鎖定與課程選項為空。
- 實習設定與個人條件仍使用產品預設值。
- 候選課程空狀態會引導使用者前往「匯入／新增」分頁搜尋政大課程、上傳截圖或手動新增。

### 既有使用者

當同一個 origin 的 `localStorage` 已有可解析的排課狀態時，沿用現有資料遷移流程：

- 從既有官方內建資料與保存的新增／刪除紀錄重建候選清單。
- 復原已選課程、出席方式、班別／教師時段、鎖定、實習設定、條件與待確認課程。
- 不改寫或主動清除舊資料。

資料以瀏覽器 origin 隔離；GitHub Pages 與其他正式站網址不會共享 `localStorage`。此限制需在教學中心說明。

## 政大課程庫搜尋

### 入口與互動

在「匯入／新增」分頁增加「政大 115-1 課程庫」區塊，與 AI 截圖匯入、手動新增並列。區塊包含：

- 單一搜尋欄，可輸入課名、教師或九碼課號。
- 搜尋按鈕；送出時顯示載入狀態並防止重複送出。
- 結果清單，顯示課名、教師、課號、學分、政大節次、限制摘要與官方課綱連結。
- 每筆結果提供「加入候選」按鈕；已存在於候選清單的課程顯示「已加入」並停用。

搜尋詞去除前後空白後不可為空。搜尋失敗時保留原輸入，並顯示「政大課程資料暫時無法查詢，請稍後重試」；零結果顯示可行的改寫建議，不清除既有結果前的候選課程。

### 資料轉換

沿用並公開現有政大資料轉換邏輯：

- 只接受學年 `115`、學期 `1` 且有九碼課號、課名與有效學分的資料。
- 把 `一二三四五六日` 與政大節次代號轉成課表會議時段。
- 把官方備註、修課限制與年級訊息整合成限制文字。
- 遇到「僅限、限、須、需、先修、雙主修、輔系、不得、優先」等限制時，建立對應的必要選課條件，預設為未勾選。
- 候選課程保留官方課綱 URL 與原始節次文字。
- 以九碼課號判斷重複，不依賴來源或內部 id，因此 AI 截圖匯入與官方搜尋不會加入同一門課兩次。

政大搜尋不使用 AI，也不需要或傳送使用者的 Gemini API Key。

## 清空候選課程

將頂部既有「恢復建議方案」改為「清空候選課程」，避免重新載入作者的內建清單。使用者確認後：

- 清空全部候選課程。
- 同時清空已排課程、鎖定課程與課程班別／教師時段選項。
- 保留個人條件、實習設定、AI 表單文字與教學狀態。
- 顯示完成訊息，並重新呈現空狀態入口。

單門課的「刪除」行為維持不變：從候選清單刪除時，也會從課表與鎖定清單移除該課。

## 元件與模組邊界

- `planner-core.mjs`：新增純函式，處理大量清除並回傳新的候選、課表、鎖定與課程選項狀態。
- `nccu-course-adapter.mjs`：負責官方 URL、回應正規化、節次解析、限制轉換與候選課程建立；不操作 DOM。
- `app.mjs`：負責初始／復原狀態、搜尋互動、結果渲染、加入候選、重複檢查與清空確認。
- `index.html` 與 `styles.css`：提供搜尋介面、載入／錯誤／空狀態，以及手機可操作的至少 44px 點擊區域。

## 測試策略

依垂直切片執行 Red → Green → Refactor，一次只新增一個失敗測試：

1. 新訪客在沒有保存狀態時得到空白候選與空白課表；既有保存狀態仍可復原。
2. 大量清除純函式會清空候選、課表、鎖定與課程選項，但不碰其他個人設定。
3. 政大節次與限制資料能轉成可排課候選課程及必要選課條件。
4. 搜尋結果以九碼課號辨識跨來源重複課程。
5. HTML 與事件契約包含搜尋、加入、載入／錯誤狀態及清空入口。
6. 真實政大 115-1 契約測試驗證搜尋回應仍包含課號、課名、學分、節次與 CORS 可用性。
7. 完整測試、建置與瀏覽器流程驗證：新訪客空白、官方搜尋、加入候選、排入課表、保存後重新整理、清空候選。

## 完成標準

- 無保存資料的新瀏覽器不會看到作者的 23 門課或預排課表。
- 原本已保存資料的瀏覽器重新整理後仍保有資料。
- 使用者可在 GitHub Pages 公開站直接搜尋 115-1 政大課程並加入候選清單。
- AI 截圖匯入與手動新增維持可用。
- 清空候選後候選、課表、鎖定與課程選項均為空，其他個人設定仍在。
- 全部自動測試與真實邊界測試通過，且手機與桌面瀏覽器流程可完成。
